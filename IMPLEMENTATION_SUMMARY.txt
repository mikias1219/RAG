================================================================================
KEY ARCHITECTURE DECISIONS
================================================================================

1. Clean Architecture + Ports & Adapters:
   - aiService, searchService, storageService are interfaces
   - Azure implementations are pluggable (can swap for AWS, GCP, local)
   - No tight coupling to Azure SDKs in business logic

2. Stateless Backend:
   - All state in PostgreSQL or Azure services
   - Horizontal scaling via Container Apps (min 1, max 3+ replicas)
   - No session affinity needed

3. Multi-Tenancy:
   - All queries filter by tenantId
   - Row-level security at query level
   - Supports sharding by tenantId in future

4. Error Handling & Resilience:
   - Retry logic with exponential backoff
   - Graceful degradation (cache failure doesn't block RAG)
   - Proper HTTP status codes (400, 401, 404, 409, 413, 500)

5. Scalability:
   - Pagination for large datasets
   - Caching for query results (30s TTL)
   - Batched Azure AI Search upserts (1000 per batch)
   - Database indexes optimized for access patterns

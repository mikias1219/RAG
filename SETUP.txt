AI102 RAG SaaS — SETUP INSTRUCTIONS

==============================================================================
LOCAL DEVELOPMENT SETUP
==============================================================================

Prerequisites:
  - Node.js 22+
  - Docker & Docker Compose
  - Azure CLI (for eventual Azure deployment)
  - PostgreSQL (via Docker)

1. Start PostgreSQL:
   docker run --name ai102-postgres \
     -e POSTGRES_USER=postgres \
     -e POSTGRES_PASSWORD=postgres \
     -e POSTGRES_DB=ai102 \
     -p 5432:5432 \
     -d postgres:16-alpine

2. Backend Setup:
   cd backend
   cp .env.example .env
   npm install
   npm run prisma:generate
   npm run prisma:migrate -- --name init
   npm run seed
   npm run dev

   Backend runs on http://localhost:8080

3. Frontend Setup (separate terminal):
   cd frontend
   cp .env.example .env.local
   npm install
   npm run dev

   Frontend runs on http://localhost:3000

4. Test:
   curl http://localhost:8080/api/health
   Visit http://localhost:3000

==============================================================================
DOCKER COMPOSE (All services + PostgreSQL)
==============================================================================

1. Set environment file (optional, for Azure credentials):
   cp infra/azure/variables.example.env .env

2. Start all services:
   docker-compose up --build

3. Access:
   Frontend: http://localhost:3000
   Backend:  http://localhost:8080/api/health
   PostgreSQL: localhost:5432 (user: postgres, password: postgres)

==============================================================================
AZURE DEPLOYMENT
==============================================================================

Prerequisites:
  - Azure subscription with access to OpenAI
  - Azure CLI installed and authenticated (az login)
  - jq installed

1. Full Deployment (creates all resources):
   chmod +x infra/azure/*.sh
   cd infra/azure
   ./deploy-all.sh eastus

   This will:
   - Create Resource Group: AI102
   - Create Azure Container Registry
   - Create Key Vault
   - Create Blob Storage
   - Create PostgreSQL Database
   - Create AI Search Service
   - Create OpenAI Service
   - Setup Container Apps Environment
   - Deploy Backend Container App
   - Deploy Frontend Container App
   - Setup Application Insights

2. Manually (step by step):
   ./00_prereqs.sh
   ./01_group_create.sh eastus
   ./02_acr_create.sh
   ./03_keyvault_create.sh
   ./04_storage_create.sh
   ./05_postgres_create.sh
   ./06_aisearch_create.sh
   ./07_openai_create.sh
   ./08_containerapps_env_create.sh
   ./09_containerapps_deploy_backend.sh
   ./10_containerapps_deploy_frontend.sh
   ./11_monitor_setup.sh

3. Configure Secrets in Key Vault:
   KV_NAME=$(cat infra/azure/outputs/keyvault_name.txt)
   
   az keyvault secret set \
     --vault-name $KV_NAME \
     --name DATABASE-URL \
     --value "postgresql://postgres:<password>@<fqdn>:5432/ai102?schema=public"
   
   az keyvault secret set \
     --vault-name $KV_NAME \
     --name AZURE-STORAGE-CONNECTION-STRING \
     --value "<connection-string>"
   
   az keyvault secret set \
     --vault-name $KV_NAME \
     --name AZURE-OPENAI-ENDPOINT \
     --value "https://<resource>.openai.azure.com/"
   
   az keyvault secret set \
     --vault-name $KV_NAME \
     --name AZURE-OPENAI-API-KEY \
     --value "<api-key>"
   
   az keyvault secret set \
     --vault-name $KV_NAME \
     --name AZURE-AI-SEARCH-ENDPOINT \
     --value "https://<service>.search.windows.net"
   
   az keyvault secret set \
     --vault-name $KV_NAME \
     --name AZURE-AI-SEARCH-API-KEY \
     --value "<admin-key>"

4. Deploy Database Schema:
   npm run prisma:deploy \
     --provider postgresql \
     --url "postgresql://postgres:<password>@<fqdn>:5432/ai102"

5. Test Deployed Application:
   FRONTEND_URL=$(cat infra/azure/outputs/frontend_fqdn.txt)
   BACKEND_URL=$(cat infra/azure/outputs/backend_fqdn.txt)
   
   curl https://$BACKEND_URL/api/health
   Open https://$FRONTEND_URL in browser

==============================================================================
BUILD & PUSH DOCKER IMAGES
==============================================================================

1. Login to Azure Container Registry:
   ACR_NAME=$(cat infra/azure/outputs/acr_name.txt)
   az acr login --name $ACR_NAME

2. Build Backend:
   docker build -t $ACR_NAME.azurecr.io/backend:latest ./backend
   docker push $ACR_NAME.azurecr.io/backend:latest

3. Build Frontend:
   docker build -t $ACR_NAME.azurecr.io/frontend:latest ./frontend
   docker push $ACR_NAME.azurecr.io/frontend:latest

==============================================================================
TROUBLESHOOTING
==============================================================================

Backend won't connect to Postgres:
  - Check DATABASE_URL in .env
  - Verify postgres service is running: docker ps
  - Test connection: psql $DATABASE_URL

Frontend API calls fail:
  - Check NEXT_PUBLIC_API_BASE_URL
  - Verify backend is running on :8080
  - Check browser console for errors

Docker build fails:
  - Clear Docker cache: docker system prune
  - Rebuild: docker-compose build --no-cache

Azure deployment fails:
  - Verify az login: az account show
  - Check resource group exists: az group show -n AI102
  - View logs: az containerapp logs show -n <app-name> -g AI102

==============================================================================
SCALING & PRODUCTION
==============================================================================

1. Configure Container Apps replicas:
   az containerapp update \
     --name ai102-backend \
     --resource-group AI102 \
     --min-replicas 2 \
     --max-replicas 10

2. Enable PostgreSQL connection pooling:
   az postgres flexible-server parameter set \
     --server-name ai102-postgres \
     --resource-group AI102 \
     --name max_connections \
     --value 500

3. Configure CDN for frontend (optional):
   az cdn profile create --name ai102-cdn -g AI102

4. Setup auto-scaling alerts:
   az monitor metrics alert create \
     --name ai102-high-cpu \
     --resource-group AI102 \
     --condition "avg Percentage Cpu > 80"

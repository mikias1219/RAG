QUICK START (3 MINUTES)
═══════════════════════════════════════════════════════════════════════════════

LOCAL DEVELOPMENT:

  1. Start PostgreSQL:
     docker run --name ai102-postgres \
       -e POSTGRES_PASSWORD=postgres \
       -p 5432:5432 -d postgres:16-alpine

  2. Backend (Terminal 1):
     cd backend
     npm install
     npm run prisma:migrate -- --name init
     npm run dev
     # Backend at http://localhost:8080

  3. Frontend (Terminal 2):
     cd frontend
     npm install
     npm run dev
     # Frontend at http://localhost:3000

  4. Test:
     curl http://localhost:8080/api/health
     Open http://localhost:3000

DOCKER COMPOSE (ALL-IN-ONE):

  docker-compose up --build
  # Frontend: http://localhost:3000
  # Backend: http://localhost:8080/api/health

AZURE DEPLOYMENT (ONE COMMAND):

  chmod +x infra/azure/*.sh
  cd infra/azure && ./deploy-all.sh eastus
  # All resources created in AI102 resource group
  # Deployment outputs in infra/azure/outputs/

API ENDPOINTS
═══════════════════════════════════════════════════════════════════════════════

GET /api/health
  • Response: { ok: true, service: string, time: ISO8601 }
  • Used for: Liveness checks, monitoring

GET /api/documents?page=1&pageSize=25
  • Response: { page, pageSize, total, items: Document[] }
  • Query params: page (default 1), pageSize (default 25)

POST /api/documents/upload
  • Content-Type: multipart/form-data
  • Body: file (single file, text/plain or application/json)
  • Response: { documentId, blobUrl, chunksIndexed }
  • Process: Upload → Blob → Chunk → Embed → Index

POST /api/chat/ask
  • Content-Type: application/json
  • Body: { question: string, sessionId?: string }
  • Response: { sessionId, answer, sources: [] }
  • Process: Embed → Retrieve → Augment → Chat → Respond

═══════════════════════════════════════════════════════════════════════════════
DATABASE SCHEMA (PostgreSQL)
═══════════════════════════════════════════════════════════════════════════════

Document
  ├── id (UUID)
  ├── tenantId (String) [Index: tenantId, createdAt DESC]
  ├── filename, contentType, blobUrl, sizeBytes
  └── createdAt (DateTime)

Chunk
  ├── id (UUID)
  ├── tenantId (String)
  ├── documentId (UUID) [FK → Document]
  ├── chunkIndex (Int) [Unique: tenantId, documentId, chunkIndex]
  ├── text (String)
  ├── tokenCountApprox (Int)
  └── createdAt (DateTime)

ChatSession
  ├── id (UUID)
  ├── tenantId (String)
  ├── userId (String) [Index: tenantId, userId, createdAt DESC]
  └── createdAt (DateTime)

ChatMessage
  ├── id (UUID)
  ├── tenantId (String)
  ├── sessionId (UUID) [FK → ChatSession (cascade)]
  ├── role (String: 'user' | 'assistant')
  ├── content (String)
  └── createdAt (DateTime)

SUPPORT & TROUBLESHOOTING
═══════════════════════════════════════════════════════════════════════════════

Backend won't start:
  → Check DATABASE_URL in .env
  → Verify PostgreSQL is running
  → Run: npm run prisma:generate

Frontend can't reach backend:
  → Check NEXT_PUBLIC_API_BASE_URL
  → Verify backend is on :8080
  → Check browser console for CORS errors

Docker build fails:
  → Clear cache: docker system prune
  → Rebuild: docker-compose build --no-cache

Azure deployment fails:
  → Verify: az login && az account show
  → Check: az group show -n AI102
  → View logs: az containerapp logs show -n <app> -g AI102
